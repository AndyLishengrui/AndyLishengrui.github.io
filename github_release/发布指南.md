# 🚀 圣经学习网站 GitHub Pages 发布指南

## 📋 快速发布步骤

### 第一步：在 GitHub 创建仓库
1. 登录 [GitHub](https://github.com)
2. 点击右上角 "+" → "New repository"
3. 仓库名建议：`bible-study-website`
4. 设为 **Public**（重要：免费版 Pages 需要公开仓库）
5. **不要**勾选 "Add a README file"（我们已经准备好了）
6. 点击 "Create repository"

### 第二步：上传代码到 GitHub
```bash
# 进入发布目录
cd github_release

# 运行一键发布脚本
./publish_to_github.sh
```

按提示输入：
- 你的 GitHub 用户名
- 仓库名（如：bible-study-website）
- Git 用户名和邮箱（如果首次使用）

### 第三步：启用 GitHub Pages
1. 访问你的仓库页面：`https://github.com/你的用户名/bible-study-website`
2. 点击 **"Settings"** 选项卡
3. 在左侧菜单找到 **"Pages"**
4. 在 Source 部分：
   - 选择 **"Deploy from a branch"**
   - Branch 选择 **"main"**
   - Folder 选择 **"/ (root)"**
5. 点击 **"Save"**

### 第四步：访问你的网站
GitHub 会自动生成网站地址：
**`https://你的用户名.github.io/bible-study-website/`**

⏰ **首次部署可能需要 5-10 分钟**

## 🔧 手动发布（如果脚本失败）

如果自动脚本有问题，可以手动执行：

```bash
cd github_release

# 初始化 Git
git init
git branch -M main

# 配置 Git（替换为你的信息）
git config user.name "你的用户名"
git config user.email "你的邮箱"

# 添加和提交文件
git add .
git commit -m "Initial commit: Bible study website"

# 添加远程仓库（替换为你的实际地址）
git remote add origin https://github.com/你的用户名/bible-study-website.git

# 推送到 GitHub
git push -u origin main
```

## ⚡ 自定义域名（可选）

如果你有自己的域名：

1. 在 `github_release` 目录创建 `CNAME` 文件：
```bash
echo "bible.yourdomain.com" > CNAME
git add CNAME
git commit -m "Add custom domain"
git push
```

2. 在域名服务商设置 CNAME 记录：
   - 名称：`bible`（或你想要的子域名）
   - 值：`你的用户名.github.io`

## 🆘 常见问题

### Q: 网站显示 404
A: 检查仓库是否为 Public，Pages 是否已启用

### Q: 推送失败
A: 确保 GitHub 仓库已创建，用户名/仓库名正确

### Q: 编辑功能不工作
A: GitHub Pages 是静态托管，编辑功能已在发布版中禁用

### Q: 网站更新不显示
A: GitHub Pages 可能有缓存，等待几分钟或强制刷新页面
