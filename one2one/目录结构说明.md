## 🎯 文件命名规范

### HTML文件
```
courses/one2one_C{课程号}_S{节号}.html
```

**示例：**
- `courses/one2one_C1_S1.html` - 第1课第1节
- `courses/one2one_C2_S1.html` - 第2课第1节
- `courses/one2one_C7_S2.html` - 第7课第2节

### JSON文件
```
data/answers/one2one_C{课程号}_S{节号}.json
```

**示例：**
- `data/answers/one2one_C1_S1.json` - 第1课第1节答案
- `data/answers/one2one_C2_S1.json` - 第2课第1节答案

## 🔗 路径引用规范

### 从index.html引用课程页面
```html
<a href="courses/one2one_C1_S1.html">第1课第1节</a>
```

### 从课程页面引用JSON数据
```javascript
const ANSWER_JSON_FILE = '../data/answers/one2one_C1_S1.json';
```

### 从课程页面返回首页
```html
<a href="../index.html">返回首页</a>
```

### 课程页面之间的导航
```html
<!-- 同一课程内 -->

<a href="one2one_C1_S1.html">上一节</a>

```

## 📝 维护说明

### 添加新课程
1. 创建HTML文件：`courses/one2one_C{X}_S{Y}.html`
2. 创建JSON文件：`data/answers/one2one_C{X}_S{Y}.json`
3. 更新`index.html`添加课程卡片
4. 确保使用相对路径

### 修改现有课程
1. 直接编辑对应的HTML文件
2. 如果修改经文，同步更新JSON文件
3. 保持文件命名规范

### 批量操作
使用提供的Python脚本进行批量操作，确保一致性。

## 🎯 设计原则

1. **简洁性**：根目录只保留index.html，课程文件统一放入courses文件夹
2. **规范性**：文件命名遵循统一规范，易于理解和维护
3. **可移植性**：全部使用相对路径，方便部署到任何服务器
4. **可扩展性**：清晰的目录结构便于后续添加新课程
